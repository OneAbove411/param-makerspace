Email Deliverability Features

Best Practices Implemented

  • Proper XHTML 1.0 DOCTYPE and meta tags for wide client support
  • Table-based layout (not div-based) for Outlook compatibility
  • Color scheme meta tags for dark mode support
  • Preheader text included (hidden preview snippet)
  • High text-to-image ratio (text only, no images)
  • Clear sender identity in footer with company name and location
  • Professional language avoiding spam trigger words
  • Proper mso conditionals for Outlook button rendering
  • All links use Supabase template variables
  • Fallback text links for clients that don't render buttons
  • "Why received" explanation specific to each template
  • Security notes where appropriate (magic link, reset password)
